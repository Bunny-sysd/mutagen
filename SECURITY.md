# Security Policy (SECURITY.md)

## Supported Versions

We actively monitor and support security fixes for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 2.x     | :white_check_mark: |
| < 2.0   | :x:                |

---

## Reporting a Vulnerability

As a security-oriented framework, the security of Mutagen and the safety of the systems running it are of paramount importance. 

If you discover a security vulnerability in Mutagen (e.g., sandbox escapes, credential leakage vectors, command injections, or path traversal issues), **please do not open a public issue.** 

Instead, report the issue responsibly by following these steps:

1. **Open a Private Security Advisory**: Please report the vulnerability privately by opening a Private Security Advisory directly on the GitHub repository page (navigate to the **Security** tab -> **Advisories** -> click **Report a vulnerability**). This keeps the details secure and lets us collaborate on a fix privately.
2. **Responsible Disclosure**: We request that you give us a reasonable period of time to review, patch, and release a fix for the vulnerability before disclosing it publicly to the community.


---

## Our Response Timeline

Upon receiving a valid vulnerability report, we will:
* Acknowledge receipt of the report within **48 hours**.
* Investigate the root cause and provide a tentative timeline for a security patch.
* Notify you when the patch is ready for review or has been merged and released.
* Give proper attribution to the security researcher in our release notes (unless you request to remain anonymous).
